A. Vision Model Selection & Rationale
Edge-computing surveillance environments enforce severe constraints on computational overhead. Relying on cloud-based inference introduces unacceptable network latency and single points of failure. Consequently, the SENTINEL architecture natively deploys the Ultralytics YOLOv8s (Small) topology via the ONNX runtime on the local edge-coordinator. YOLOv8s abandons traditional anchor-based detection in favor of an anchor-free, decoupled head approach. This architectural shift drastically reduces parameter count and spatial computational complexity. We benchmarked multiple topologies; while heavier models yielded a marginal 2-3% increase in Mean Average Precision (mAP), the corresponding drop in frame-throughput destroyed real-time tracking viability. Implementing the ONNX execution provider allows the system to bypass Python's Global Interpreter Lock (GIL) overhead during raw tensor multiplication. This maximizes CPU clock cycles without requiring specialized GPU silicon.

B. Asynchronous Edge-Inference Pipeline
A fundamental bottleneck in distributed video surveillance is network buffer saturation. When an ESP32-CAM module transmits an MJPEG stream over standard Wi-Fi protocols, decoding delays inherently cause rapid frame queuing. Subjecting the inference engine directly to this queued stream forces the AI to process outdated visual telemetry. To solve this, the system isolates video ingestion from tensor processing through an asynchronous Producer-Consumer threading model. A dedicated background thread continuously drains the MJPEG socket, aggressively overwriting a thread-safe mutex lock with only the most recent frame. Simultaneously, the primary inference loop pulls directly from this lock. This decoupling guarantees the YOLOv8s model evaluates frame data with absolute zero queuing latency. Despite network fluctuations, the pipeline reliably sustains 12 to 15 FPS execution under standard edge-hardware limits.

C. Algorithmic False-Positive Mitigation
Raw machine learning outputs are inherently noisy. In high-stakes command environments, false-positive threat classifications result in critical alarm fatigue. To stabilize the output vector, we designed a temporal hysteresis algorithm. Instead of evaluating frames in isolation, the system forces a sliding temporal window over the last five inference results. A classification is only verified and pushed to the Firebase Real-Time Database if bounding-box confidence strictly exceeds the 70% threshold across at least three consecutive physical frames. This temporal suppression mathematically annihilates transient classification spikes—such as environmental shadows or camera hardware artifacts temporarily triggering the loss function. It ensures that only verified, persistent physical presence alters the deployed threat matrix.
